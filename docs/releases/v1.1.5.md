# V1.1.5

发布日期：2026-06-30

自 V1.1.0 起的一轮后端加固 + 查询能力增强 + 启动脚本收敛。**逐项修改、根因与决策记录见 [logs/](../../logs/)**，本说明只列高层变化。

## 重点变化

**正确性 / 安全**

- 修复后台编辑后全文搜索错位（FTS 改按唯一键 `code` 连接）、详情接口返回陈旧缓存。
- CORS 收紧为按 config 解析的白名单（绝不返回 `*`）；会话改 HMAC-SHA256 签名令牌（常数时间校验、可选过期、`Secure` 可配）。

**健壮性 / 运维**

- 后台任务历史按上限裁剪；管线步骤选择改数值比较；`has_photo` 双向同步；瓦片下载磁盘配额守护。

**查询能力**

- 新增 `GET /api/relics/facets`：当前筛选下全 9 维分面计数 + 总数 + has_3d。
- 新增 `GET /api/relics/list`：同一筛选下分页精简行。
- 厘清 `/api/relics` 定位（前端交叉筛选的合法数据源，撤销 deprecated 误标）。
- AI system prompt 按数据规模分级注入，防 Token 膨胀。

**工程 / 启动**

- 根目录启动脚本收敛为单一入口 `start-all.bat` / `start-all.sh`（跨平台 `start.py`），删除 Windows 专用的 `start-backend.bat` / `start-frontend.bat`。
- 新增编码字典单源漂移守护测试，补齐路由端点测试。后端测试套件 **121 passed**。

## 兼容性

- 后端 API 路径保持兼容；`/api/relics` 仍可用（不再标 deprecated）。
- `data/input/`、`data/output/` 数据约定与 `config.yaml` 模板流程不变。
- 启动方式变更：原 `start-backend.bat` / `start-frontend.bat` 已并入 `start-all.*`；仅起后端可直接运行 `python platform/webgis/serve.py`。
- 若开启 `server.enable_auth`，旧的静态 `authenticated` cookie 失效，需重新登录。
