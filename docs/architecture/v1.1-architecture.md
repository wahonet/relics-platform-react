# V1.1 架构说明

日期：2026-05-16
版本：1.1.0

V1.1 是一次架构调整：保留 V1.0.0 的业务能力，减少根目录入口、拆分后端大模块、整理 Admin Vue 大页面、补齐测试与 CI，并把文档归并到少量稳定入口。

## 启动边界

根目录只保留两个 BAT：

- `start-backend.bat`：启动 FastAPI 后端。
- `start-frontend.bat`：启动 React WebGIS 和 Vue Admin 两个前端开发服务。

数据管线不再作为根目录 BAT 存在。它保留在 `platform/scripts/run_pipeline.py` 和 Vue Admin 管线工作台中，按需手动执行。

端口分工：

- `8000` 是 FastAPI 后端端口，同时挂载构建后的 `/app/` 与 `/admin-ui/`。
- `5174` 是 React WebGIS 的 Vite 开发端口。
- `5173` 是 Vue Admin 的 Vite 开发端口。

开发时通常同时启动 `8000 + 5174 + 5173`；只查看构建后的集成版本时，启动后端访问 `8000/app/` 即可。

## 后端职责

- `main.py` 是 FastAPI 组合入口，负责应用生命周期、配置加载、静态目录挂载、legacy fallback 和 router 组装。
- `tile_routes.py` 负责瓦片代理、离线瓦片下载、缓存统计和后台瓦片概览。
- `terrain_routes.py` 负责 DEM terrain tile API。
- `admin.py` 负责管线状态、管线详情、后台任务、DOCX 上传和单文件处理。
- `admin_task_service.py` 负责脚本注册、别名、子进程任务、日志收集和并发保护。
- `admin_relic_routes.py` 负责后台文物 CRUD、审计、批量操作、导入导出和 Dashboard 统计入口。
- `data_loader.py` 保留 `store.*` 兼容门面。
- `data_serializers.py`、`data_admin_queries.py`、`data_admin_stats.py`、`survey_coverage.py` 承接从 `data_loader.py` 拆出的可复用逻辑。

## 前端职责

React WebGIS 是主业务界面，负责地图、筛选、详情、统计、三维模型、PDF、工作日志、AI 问答和瓦片下载。

Vue Admin 是后台操作界面，负责 Dashboard、管线工作台、文物编辑、批量操作、导入导出、审计和回收站。

V1.1 对 Admin Vue 做了第一轮拆分：

- `Dashboard.vue` 保留页面结构。
- `useDashboard.ts` 承接 Dashboard 状态和动作。
- `dashboard.css` 承接 Dashboard 样式。
- `RelicEditDialog.vue` 与 `Relics.vue` 的大样式块拆入 `src/styles/`。

## 管线职责

`platform/scripts/run_pipeline.py` 是唯一命令行 orchestrator，支持：

- `--list`
- `--dry-run`
- `--skip`

默认步骤：

1. DOCX 转 Markdown。
2. Markdown 转结构化数据集。
3. 提取照片。
4. 提取图纸。
5. 工作日志转 PDF。
6. 行政边界处理。
7. 构建 SQLite 数据库。

## 测试与 CI

测试目录保留后端核心单测：

- 管线 orchestrator。
- Admin 任务服务。
- 数据 helper 和 delegate。
- Admin router 组合。

CI 位于 `.github/workflows/ci.yml`，覆盖 Python 测试、React 类型检查/构建、Vue 类型检查/构建。
